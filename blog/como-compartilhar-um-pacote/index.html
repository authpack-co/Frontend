Como compartilhar um pacote no AuthPack

Depois de montar um pacote, compartilhá-lo é o que entrega acesso a outras pessoas — sem revelar nenhuma senha. O AuthPack faz isso por links de convite, e você escolhe entre um link geral ou links únicos de uso individual.

Equipe AuthPack
Tutoriais do produto

Compartilhar um pacote significa dar a outra pessoa acesso a todas as sessões que estão dentro dele. Ela passa a usar esses acessos já logada, sem ver e-mail nem senha. Tudo acontece por um link de convite — e você mantém o controle de quem entra e quando esse acesso termina.

Pré-requisito

Você precisa de um pacote já criado e, idealmente, com pelo menos uma sessão dentro. Se ainda não tem, veja como criar seu primeiro pacote e como adicionar uma sessão.

Passo a passo

Passo 1 · Abra o pacote no painel

No painel do AuthPack, vá até a sua coleção e localize o pacote que você quer compartilhar.

Passo 2 · Clique em "Compartilhar"

No pacote, clique em Compartilhar. Abre-se a janela de compartilhamento, com as opções de convite do pacote.

Passo 3 · Escolha entre link geral e link único

O AuthPack oferece dois tipos de convite, para situações diferentes:

  • Link geral. Um convite reutilizável para o pacote — ideal quando você quer dar acesso a várias pessoas com o mesmo link (um time, uma turma, um grupo). Pode ser ativado ou desativado quando quiser.
  • Link único. Um convite de uso individual, com validade própria, perfeito para conceder acesso a uma pessoa específica e manter o controle pessoa a pessoa. Cada link único pode ser copiado e revogado de forma independente.

Passo 4 · Copie e envie o link

Copie o link escolhido e envie para quem deve receber o acesso — por qualquer canal. Quando a pessoa abrir o link, ela verá uma prévia do pacote (nome, serviços incluídos e quem está compartilhando) e poderá ativar o acesso. O passo a passo do lado de quem recebe está em como usar um acesso recebido.

Link geral é a porta da frente do time; link único é uma chave nominal. Use o primeiro para grupos e o segundo quando quiser conceder e cortar acesso pessoa por pessoa.

Mantendo o controle

  • Desative o link geral para impedir novas ativações a qualquer momento.
  • Revogue um link único para cortar o acesso de uma pessoa específica.
  • Nada disso exige trocar senha — porque a senha nunca foi compartilhada. Veja como revogar um acesso compartilhado.

Perguntas frequentes

Qual a diferença entre o link geral e o link único?

O link geral é um convite reutilizável para convidar várias pessoas ao mesmo pacote, e pode ser ativado ou desativado. O link único é de uso individual, com validade própria, ideal para conceder acesso a uma pessoa específica.

Quem recebe o link vê a minha senha?

Não. O link dá acesso à sessão do pacote, não à credencial. Quem ativa o convite passa a usar os acessos já logado, sem nunca ver a senha.

Posso parar de compartilhar um pacote depois?

Sim. Você pode desativar o link geral ou revogar um link único a qualquer momento. O acesso é cortado sem precisar trocar nenhuma senha.
